# Comprehensive Documentation of the Quality Management System (QMS) Platform

This document contains a complete and comprehensive explanation of every aspect of the website, including its design, functionality, and section layout.

---

## 1. Introduction and Website Objective
The website is a digital platform for managing a Quality Management System (QMS) compliant with the ISO 9001:2015 standard. Its primary objective is to automate audit processes, record tracking, ensure compliance with international standards, and facilitate access to Standard Operating Procedures (SOPs).

---

## 2. General Structure and Section Layout
The website is divided into two main parts:

1. **Sidebar**: Always located on the left, it contains all essential links. It can be collapsed to save space.

2. **Main Content**: This changes depending on the section you are browsing and always includes a **Header** containing the search bar, notifications, and user information.

### Sidebar Contents:

- **Dashboard**: System overview.

- **Projects**: Track quality records for each individual project.

- **Audit Dashboard**: The heart of the system, where records are reviewed, accepted, or rejected.

- **Archive**: Access archived records.

- **Risk Management**: Monitor quality risks.

- **Activity**: A complete log of all transactions performed on the system.

- **Procedures**: View and modify Standard Operating Procedures (SOPs).

- **ISO Manual**: Digital manual of ISO specifications.

- **Admin Accounts**: (Visible only to administrators) For managing users and their permissions.

---

## 3. Main Dashboard
This is the first page you see after logging in. Its design is based on a colored card system:

- **Hero Section**: Welcomes the user and displays the synchronization status with Google Sheets, along with a Sync button for real-time data updates.

- **Top Metrics**: Five colored cards:

- **Evidence**: Total files uploaded.

- **Approved**: Files that have been reviewed and accepted.

- **Pending**: Files awaiting review.

- **Gaps**: Forms for which no files have been uploaded yet.

- **Rejected**: Files with issues that require modification.

- **System Modules**: A grid containing sections such as Sales, Operations, Quality, HR, and R&D. Each card displays its corresponding ISO standard and the number of completed forms.

- **Side Panel (Widgets)**:

- **Quick Actions**: Quick buttons for uploading files or writing reports.

- **Audit Readiness**: A progress bar showing the system's readiness for external auditing.

---

## 4. Audit Dashboard
This is the section for auditors and quality control personnel:

- **Filters**: You can search by code, name, category, project, or even year.

- **Tabs**:

- **Pending**: All new files that need review.

- **Approved**: Accepted files (shown in green).

- **Issues**: Rejected files (shown in red) with the reason.

- **Overdue**: Forms that are overdue for updates.

- **Never Filled**: Forms that have never been used.

- **Automated Audit**: The Run Audit button opens a window that prompts the system to automatically check all files for missing data or dates.

- **Bulk Actions**: You can select a group of files and accept or reject them with a single click.

---

## 5. Record Details Page
Clicking on any record opens a dedicated page:

- **Header**: Displays the record code (e.g., P-SLS-01), its name, and its classification.

- **Links**: Direct buttons to open the blank form or the record folder in Google Drive.

- **Technical Specs**: A table showing (Who is responsible? How often is it updated? Where is it saved? How long is it saved for?).

- **Review Panel**: Here, the auditor writes their feedback and changes the status to "Approved" or "Rejected".

---

## 6. Procedures Section
This section is dedicated to written Standard Operating Procedures (SOPs):
- **Digital**: Displays procedures as digital text that can be read and edited directly within the website.

- **Drive Archive**: A window that displays PDF and Word files stored on Google Drive directly within the website without needing to leave it.

---

## 7. Admin Page (Admin Accounts & Settings)
This is the "Admin Editor" you requested:

- **User Management**: A table showing all employees registered with the system.

- **Admin Edit Data**: Clicking on any user opens a panel that allows the admin to change:

- Name and email address.

- **Role**: (Admin, Manager, Auditor, User).

- **Status**: Activate or deactivate your account.

- **Password**: Send a password reset link.

- **Pending Approvals**: A special section showing newly registered employees awaiting admin approval to access the system.

---

## 8. Forms and Record Format
- **Templates**: Blank forms that are uploaded and filled out.

- **Records**: Completed copies uploaded to the system. They appear on the website as "cards" or "rows in a table" containing a file icon, upload date, and the uploader's name.

---

## 9. Additional Features (Where to Find Everything?)
- **Search**: Located at the top of almost every page.

- **Sync**: The circular (Refresh) button is always available.
